# Find Yourself

`Find Yourself` 是一款 AI 求职伴侣，帮助用户从“我是谁、我想做什么、我能做什么”走到“投递、面试、复盘、调整”，把求职过程变成一条可持续推进的旅程。

## 1. 产品概述

Find Yourself 不是单纯的简历工具，而是一个围绕求职全流程设计的陪伴式产品。它同时解决三类问题：

1. 方向问题：我适合什么岗位？我想做什么？
2. 匹配问题：我的简历和 JD 哪些地方对得上，哪些地方有缺口？
3. 过程问题：我怎么把分析结果变成行动，并持续复盘？

---

## 2. 目标用户

### 2.1 主要用户

- 正在找工作的应届生
- 正在转方向的职场人
- 想优化简历和面试表现的求职者
- 容易在投递和等待过程中焦虑的人

### 2.2 用户特征

- 对岗位方向不够确定
- 简历内容有，但表达不够聚焦
- 想知道自己和目标岗位的差距
- 需要持续陪伴，而不是一次性答案

---

## 3. 核心场景

### 场景 A：我不知道该投什么
用户有经历，但不确定适合什么岗位，希望先得到方向建议。

### 场景 B：我有简历，但不知道怎么改
用户已经有简历和目标 JD，希望系统帮他做匹配分析、找缺口、给优化建议。

### 场景 C：我想冲刺面试
用户已经开始投递，需要模拟面试题、回答建议和重点准备方向。

### 场景 D：我想持续推进而不是躺平
用户希望把建议拆成行动项，持续记录复盘，避免求职过程失控。

### 场景 E：我现在很焦虑
用户需要一个安全入口表达压力、挫败感和不确定性。

---

## 4. 产品目标

- 帮助用户更快明确方向
- 提高简历与岗位的匹配效率
- 降低求职过程中的焦虑和失控感
- 让每一次分析都能落到行动和复盘

## 6. 功能清单

### 6.1 模型配置与校验

**能力说明**
- 填写 API Key、Base URL、模型名
- 验证模型连通性
- 支持本地记住密钥或仅会话保存

**价值**
- 降低首次接入门槛
- 保证后续分析能力可用

### 6.2 需求交流

**能力说明**
- 收集用户当前的困惑、目标和限制
- 将需求沉淀到本地草稿

**价值**
- 帮助用户先把问题说清楚
- 为后续分析提供上下文

### 6.3 简历与 JD 输入

**能力说明**
- 粘贴简历全文
- 粘贴目标 JD
- 支持 PDF / DOCX 上传

**价值**
- 为匹配分析、改写和面试准备提供原始输入

### 6.4 职业方向与行动计划

**能力说明**
- 根据简历和偏好分析适合方向
- 给出阶段性行动建议

**价值**
- 帮助用户从“迷茫”转向“可执行”

### 6.5 岗位匹配分析

**能力说明**
- 对比简历与 JD
- 输出匹配评分、缺口分析与优化建议

**价值**
- 让用户知道“该补什么、先改什么”

### 6.6 多 JD 对比

**能力说明**
- 对多个岗位横向比较
- 给出投递优先级建议

**价值**
- 提高投递决策效率

### 6.7 ATS 检查

**能力说明**
- 检查简历与岗位关键词覆盖情况
- 提醒结构和表达上的风险点

**价值**
- 提高简历通过筛选的概率

### 6.8 简历润色与改写对比

**能力说明**
- 根据目标岗位调整表达方式
- 展示改写前后的差异

**价值**
- 让用户看见改动的方向和收益

### 6.9 面试模拟

**能力说明**
- 按岗位生成面试题
- 给出准备建议和回答框架

**价值**
- 提前建立面试感
- 帮助用户减少临场慌乱

### 6.10 求职旅程与复盘

**能力说明**
- 记录行动项
- 保存复盘日志
- 追踪当前阶段推进情况

**价值**
- 让求职从一次次焦虑变成可积累的过程

### 6.11 画像记忆与情绪记录

**能力说明**
- 保存稳定信息、偏好和待确认问题
- 记录情绪、压力和阶段性体会

**价值**
- 让系统更理解用户
- 提供更持续的陪伴感

## 7. 用户流程

### 标准流程

1. 打开 Landing Page，了解产品
2. 配置模型并完成校验
3. 输入自己的困惑和目标
4. 粘贴简历与 JD
5. 生成方向、匹配、润色、面试建议
6. 将建议拆成行动项
7. 持续复盘和记录情绪

### 快速启动流程

- 直接进入 `setup.html`
- 完成配置后进入 `chat.html`
- 再进入 `resume.html` 与 `analyze.html`

## 11. 产品原则

- 先理解人，再理解岗位
- 先给方向，再给细节
- 先给行动，再给评判
- 先陪伴，再优化

本项目的目标不是一次性输出标准答案，而是持续帮助用户从模糊走向清晰，从清晰走向行动，再从行动走向复盘。
